# 🐾 Лагерные питомцы

Веб-система мотивации для летнего лагеря. Дети (6–11 лет) заводят себе
виртуального питомца, который живёт на общем сайте. Преподаватель выдаёт
очки и шмотки за успехи на занятиях — питомцы прокачиваются, в галерее
виден лидерборд.

**Девочке/мальчику нужен только браузер:** сайт открывается с телефона,
планшета, ноутбука по адресу `https://USERNAME.github.io/pets`.

---

## ✨ Что внутри

- **Галерея** (`index.html`) — все питомцы лагеря, сортировка по очкам (лидерборд) или по имени.
- **Регистрация/вход** (`login.html`) — ребёнок выбирает вид, придумывает кличку и PIN.
- **Страница питомца** (`pet.html`) — питомец со шмотками, кнопка «покормить», инвентарь.
- **Админка** (`admin.html`, десктоп) — ты выставляешь очки, выдаёшь шмотки, добавляешь виды.

**Как это работает технически:**
Сайт — это статика на GitHub Pages. Все данные (дети, очки, инвентарь, виды)
лежат в одном файле `data/data.json` в репозитории. Браузер читает и пишет
его через **GitHub Contents API**. Запись = коммит в репо, поэтому вся
история изменений видна в `Commits` — это встроенный аудит.

## 🚀 Деплой за 15 минут

### 1. Создай репозиторий на GitHub
- Зайди на [github.com/new](https://github.com/new).
- Имя репозитория: например `pets` (тогда адрес будет `USERNAME.github.io/pets`).
- Тип: **Public** (Pages на бесплатном тарифе работает только для публичных репо).
- **Не** добавляй README/.gitignore — пустой репо.
- Нажми **Create repository**.

### 2. Залей файлы проекта
В этой папке (`ZCodeProject`) открой терминал и выполни:

```bash
git init
git add .
git commit -m "init: лагерные питомцы v1"
git branch -M main
git remote add origin https://github.com/USERNAME/pets.git
git push -u origin main
```

(замени `USERNAME` на свой ник).

### 3. Сделай Personal Access Token
- GitHub → Settings → Developer settings → **Personal access tokens** → **Fine-grained tokens** → *Generate new token*.
- **Repository access:** Only select repositories → выбери `pets`.
- **Repository permissions:** Contents → **Read and write** (остальное не трогай).
- **Expiration:** поставь до конца смены (или 90 дней).
- Сгенерируй, **скопируй токен** (он начинается на `github_pat_`).

### 4. Впиши конфиг
Открой `config.js` в корне проекта и заполни:

```js
window.CONFIG = {
  owner: "USERNAME",          // твой ник на GitHub
  repo: "pets",               // имя репо
  branch: "main",
  token: "github_pat_....",   // токен из шага 3
  adminPin: "0000",           // придумай пин для входа в админку
  dataPath: "data/data.json",
  apiBase: "https://api.github.com",
};
```

Закоммить изменения:
```bash
git add config.js
git commit -m "fill config"
git push
```

> ⚠️ **Токен виден в коде страницы.** Это допустимо для лагеря (ты сам так решил):
> данных в `data.json` не жалко, умный ребёнок максимум подсмотрит чужие PIN'ы.
> Если захочешь спрятать токен — см. раздел «Cloudflare Worker» ниже.

### 5. Включи GitHub Pages
- В репо: **Settings → Pages**.
- **Source:** Deploy from a branch.
- **Branch:** `main` / `(root)` → **Save**.
- Через 1–2 минуты сайт будет жить по адресу `https://USERNAME.github.io/pets`.

### 6. Проверь
- Открой сайт. Должна загрузиться галерея с 4 демо-детьми (Аня, Боря, Вера, Глеб).
- Зайди в админку (кнопка ⚙ справа сверху), введи `adminPin` из конфига.
- Поменяй очки кому-нибудь → проверь, что в репо появился новый коммит в `data/data.json`.

Готово! 🎉

## 🎨 Замена SVG-заглушек на свои спрайты (itch.io и т.д.)

Все спрайты лежат в `assets/sprites/`:
```
assets/sprites/
├── pets/    ← питомцы: cat.svg, dog.svg, ...
└── items/   ← шмотки: crown.svg, glasses.svg, ...
```

### Простая замена
1. Скачай спрайт-пак с itch.io (подойдут паки для пиксель-арт RPG,宠物 / pet pack и т.п.).
2. Положи свой файл в нужную папку **с тем же именем**:
   - хочешь заменить кота → `assets/sprites/pets/cat.png` (заменяет `cat.svg`).
3. В `data.json` поменяй `sprite` у соответствующего вида/предмета:
   ```json
   { "id": "cat", "name": "Кот", "sprite": "cat.png" }
   ```
4. Закоммить — рендерер подхватит новый файл автоматически.

### Конвенция для спрайтов
- **Размер:** 256×256 px (canvas). Сам персонаж — по центру.
- **Фон:** прозрачный (PNG или SVG).
- **Якоря для одежды:**
  - `hat` (голова) — рисуется в верхней трети, ~y=0–80.
  - `face` (лицо) — на уровне глаз, ~y=78–110.
  - `neck` (шея) — под головой, ~y=130–175.
  - `body` (тело/плащ) — на туловище, ~y=120–256.

Если твои спрайты другого размера — отредактируй якоря в `assets/js/pet-renderer.js`
(там слои просто накладываются, можно добавить смещения через CSS-классы).

### Добавление новых видов/шмоток
**Не обязательно трогать файлы!** В админке есть вкладки «Шмотки» и «Виды» —
добавляй новые прямо оттуда. Главное, чтобы соответствующий спрайт лежал в `assets/sprites/`.

## 🔐 Безопасность

Токен лежит в `config.js` и виден всем, кто откроет сайт. Это означает:
- Любой может читать/писать твой `data.json` (менять очки, удалять детей).
- Любой может увидеть PIN'ы детей в `data.json` напрямую.

Для лагеря это **приемлемо** по твоему решению — данные не чувствительные,
умных хакеров среди 6–11-летних не предвидится.

Если всё же захочешь спрятать токен — есть вариант с Cloudflare Worker
(см. ниже). Обёртка API в `store.js` уже готова к этому переключению.

## 🛠️ Локальный запуск для разработки

GitHub Pages работает только на готовом репо. Чтобы тестировать локально
(например, вёрстку), подними простой сервер в папке проекта:

```bash
# Python 3
python -m http.server 8000
```
Открой `http://localhost:8000`.

> Примечание: запись через GitHub API будет работать и с localhost
> (CORS у GitHub разрешён), так что можно тестировать весь поток локально,
> если `config.js` уже заполнен под реальный репо.

## ❓ Частые вопросы

**Q: А если двое детей одновременно нажмут «сохранить»?**
`store.js` перед записью перечитывает актуальный `data.json`, мержит изменения
и пишет с актуальным `sha`. Если между чтением и записью успел кто-то другой
(409 Conflict) — повторяет цикл до 3 раз. Плюс все записи сериализованы в
одну очередь. Так что гонки не страшны.

**Q: Что если ребёнок забудет свой PIN?**
Ты видишь все PIN'ы в админке (или прямо в `data.json`). Просто назови ему.

**Q: Что если ребёнок сядет с чужого телефона?**
Он выбирает себя в списке и вводит PIN. Сессия в localStorage — на чужом
телефоне логин заново, на своём не надо.

**Q: Можно ли сбросить всё и начать заново?**
Удали/отредактируй `data/data.json` прямо в репо через веб-интерфейс GitHub.

**Q: Лимиты API?**
GitHub API даёт 5000 запросов в час для авторизованных. На 30 детей и их
тысячи кликов в день — с огромным запасом.

## 🧩 Опционально: спрятать токен через Cloudflare Worker

Если решишь, что токен в коде — это слишком:

1. Зарегистрируйся на [Cloudflare](https://dash.cloudflare.com) → Workers.
2. Создай Worker с кодом-прокси (GET/PUT проксирует запросы к GitHub API,
   подставляя токен из секрета воркера).
3. В `config.js` поменяй `apiBase` на адрес воркера (`https://pets-api.твой-воркер.workers.dev`)
   и удали `token`.
4. В `store.js` ничего менять не надо — обёртка уже ходит через `CONFIG.apiBase`.

Это бесплатно (100 000 запросов/день) и работает из РФ.
